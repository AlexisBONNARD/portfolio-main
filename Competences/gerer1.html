    Apprentissages critiques :
  • AC14.01 | Mettre à jour et interroger une base de données relationnelle (en requêtes directes ou à travers une application)
  • AC14.02 | Visualiser des données
  • AC14.03 | Concevoir une base de données relationnelle à partir d’un cahier des charges

Durant ma 1ère année de BUT Informatique, j'ai eu l'opportunité de mettre en place mon apprentissage dans la conception d'une base de données de l'entreprise fictive : « Matski »

Partie 1 : Création de la base de données

Base de données actuelle de Matski

Base de données future

L’entreprise Matski m’a transmis sa base de données existante, qui présentait plusieurs incohérences et manques. Après avoir expliqué au client les problèmes identifiés dans cette base, j’ai conçu une nouvelle base de données répondant à leurs besoins actualisés.

    La base de données actuelle présentait notamment les erreurs suivantes :
  • La table Client contenait des attributs non pertinents.
  • La table Article comportait des attributs liés au contact, qui auraient dû être intégrés à la table Client.
  • La table Article incluait l’attribut PrixVente, déjà présent dans l’association TarifVente, rendant cet attribut redondant et inutile.

La nouvelle base de données conçue corrige ces erreurs et intègre également la gestion des achats de l’entreprise. Elle introduit de nouvelles tables, notamment : Fournisseur, Contact, Approvisionnement ainsi que les caractéristiques des articles.

Partie 2 : Requêtage

Au cours de l’analyse de la base de données, l’entreprise a exprimé le besoin d’obtenir certaines informations afin de réaliser une étude économique de son activité. Elle a donc formulé plusieurs demandes auxquelles j’ai répondu en réalisant différentes requêtes sur leur base de données existante.
